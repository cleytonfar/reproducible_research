As we can see from the warning messages, as the mean of steps is a float number and steps variable is an integer type, there is an implicit coercion of the float to integer inside the data.table. If we wanted not to happen this coercion, we would first transform steps variable to float and then apply the mean imputation. For the purpose of this projejct assignment, I will let it as it is.

Now lets investigate if there is any change on the activity pattern between weekdays and weekends.

First, lets create a variable called type_day that specify if that date is weekday or weekend. Then, we will plot the series for each type_day:

foo <- foo %>% 
    mutate(type_day = ifelse(weekdays(date) == "Sunday" | weekdays(date) == "Saturday", "weekend", "weekday")) %>% 
    as.data.table()

foo[, .(mean = mean(steps, na.rm = T)),  by = .(interval, type_day)] %>% 
    mutate(interval = as_datetime(interval)) %>% 
    ggplot(data = ., aes(x = interval, y = mean)) +
    geom_line() + geom_point() + 
    facet_grid(rows = vars(type_day)) + 
    scale_x_datetime(breaks = scales::date_breaks("1 hour"),
                     date_labels = "%H:%M")

From the previous plot, we can that during weekends, people do not seem to take much steps during the 6-8 hours in the morning. Also, the average steps taken has a smaller variance during the 8-21h in the weekend compared to week days.
